# web_project_portfolio_es

# 🌐 Portafolio Web - Raúl García Lemus

## 📌 Descripción del proyecto

Este proyecto es un **portafolio web personal** desarrollado para presentar el perfil profesional, habilidades técnicas y proyectos destacados de Raúl García Lemus como desarrollador web.

El sitio incluye:

- Una sección de presentación con información profesional y enlaces relevantes (CV, GitHub, contacto).
- Un apartado de habilidades con tecnologías utilizadas.
- Una sección de proyectos destacados con descripción, tecnologías y enlaces a demos en vivo.
- Un footer con información de contacto y redes sociales.

El diseño está enfocado en ser **moderno, limpio y completamente responsivo**, adaptándose a diferentes tamaños de pantalla (desktop, tablet y móvil).

---

## ⚙️ Funcionalidad principal

- 📱 Diseño responsivo con adaptación fluida entre resoluciones.
- 🎨 Interfaz visual moderna con efectos de gradientes y hover.
- 🧩 Organización modular de estilos CSS.
- 🔗 Integración con enlaces externos (GitHub, LinkedIn, demos).
- 📄 Descarga directa de CV en PDF.
- 🖼️ Visualización de proyectos con overlay y tecnologías utilizadas.

---

## 🛠️ Tecnologías y técnicas utilizadas

### Frontend

- **HTML5**
  - Estructura semántica para mejorar accesibilidad y SEO.
- **CSS3**
  - Flexbox para layout.
  - Media queries para diseño responsivo.
  - Uso de `clamp()` para escalado dinámico de tamaños.
  - Gradientes y efectos visuales personalizados.

### Arquitectura de estilos

- Separación modular de archivos CSS:
  - Estilos globales (`style.css`)
  - Estilos de contacto (`contact.css`)
  - Tipografías (`fonts.css`)
  - Breakpoints específicos (`style768.css`, `style320.css`)
  - Archivo central que importa todo (`index.css`)

### Tipografía

- Uso de fuentes personalizadas mediante `@font-face` para mayor control visual.

### UI/UX

- Diseño centrado en experiencia de usuario:
  - Navegación clara
  - Jerarquía visual bien definida
  - Feedback visual con hover effects
  - Uso de overlays en tarjetas de proyectos

### Buenas prácticas

- Código limpio y organizado
- Uso de nombres de clases descriptivos (BEM-like)
- Separación de responsabilidades (estructura vs estilos)
- Uso de assets optimizados (imágenes y fuentes)

---

## 🚀 Objetivo del proyecto

Este portafolio tiene como objetivo:

- Mostrar habilidades técnicas en desarrollo web.
- Servir como carta de presentación para reclutadores.
- Demostrar capacidad de construir interfaces modernas y responsivas.
- Consolidar conocimientos en HTML, CSS y buenas prácticas de frontend.
